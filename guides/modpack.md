# 📦 Modpack

Chertia Requiem uses a custom modpack simply called _Requiem_. The modpack is purely client-side and adds a number of QOL features. We ask that you only use the mods that are provided in this modpack, unless an exception is granted by an administrator.&#x20;

### Download

To install the modpack, we highly recommend using the Modrinth launcher, which you can download [here](https://modrinth.com/app). Once installed, navigate to the explore page and search for 'Requiem' - you should see the Chertia logo as the modpack's icon.

Alternatively, the modpack file can be [downloaded from the Modrinth page here](https://modrinth.com/modpack/requiem-vanilla). Ensure you download the latest version.

If you're averse to the Modrinth launcher, it is possible to download the .mrpack file and exchange it for a ZIP file by visiting [this website](https://fabulously-optimized.github.io/mrpack-to-zip/) and providing the Modrinth ID for the modpack, which is `AlxoR1AY`. You can then install the mod files manually - however, we strongly recommend using the launcher.

For a list of mods included, click the 'Versions' tab on the Modrinth page and click the name of the latest version (for example, 1.2.0). Most mods can be disabled if you'd prefer not to have them.

### Update

There are a few different ways to update the modpack, depending on the instance you have chosen.

#### Modrinth

If the modpack has been directly downloaded from the Modrinth app, updating is simple.

| ---------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------- |
| Select the instance in your Modrinth library.                                                                                      | <img src="../.gitbook/assets/image (13).png" alt="" data-size="original"> |
| Select the 'settings' icon on the top right of the instance, next to 'Content'.                                                    | <img src="../.gitbook/assets/image (14).png" alt="" data-size="original"> |
| Under 'Installation', select 'Change version'.                                                                                     | <img src="../.gitbook/assets/image (16).png" alt="" data-size="original"> |

| At the bottom right, select 'Update to \[version]'.                                                                                | <img src="../.gitbook/assets/image (19).png" alt="" data-size="original"> |
| Done! Allow the mods to download. Certain resource and shader packs may not be compatible with the latest version.                 | <img src="../.gitbook/assets/image (20).png" alt="" data-size="original"> |

#### Manual .mrpack file

If you installed the modpack manually in the Modrinth app using a .mrpack file, you can simply load the new .mrpack file downloaded from the [versions](https://modrinth.com/modpack/requiem-vanilla/versions) page and install it the same way. I recommend copying over your `config` folder, `options.txt`, and `xaero` folder at the very least, but you can also copy over your `shaderpacks`, `resourcepacks`, `saves`, and `screenshots` folders if you'd like.&#x20;

My actual recommendation is to take this opportunity to switch to downloading directly from Modrinth to make the updating process much easier. You can copy over the folders the same way.

#### Other Installations

Unfortunately, it's impossible for me to account for every way that the modpack might be installed. If you're using another installation, follow the initial steps to install the modpack on this page. Similar to above, I recommend copying over your `config` folder, `options.txt`, and `xaero` folder at the very least, but you can also copy over your `shaderpacks`, `resourcepacks`, `saves`, and `screenshots` folders if you'd like.&#x20;

{% hint style="warning" %}
From time to time, the modpack is updated to change out certain mods. It is not necessary for you to update, unless we are changing Minecraft versions (for example, 26.1 -> 26.2).
{% endhint %}
